STEP 1.4 — Build output parser
FILE: agent/parser.py
STATUS: [ ] Not done

  What this file does:
  - Exposes function: parse_action(text) -> (tool_name, args)
  - Uses regex to extract tool name and JSON args from
    model output like: Action: lookup_policy({"user_id": "P-1001"})
  - Returns (None, None) if no valid action found
  - Must handle messy model output gracefully

  Test command after writing:
    python agent/parser.py
  Expected output:
    Test 1: ('lookup_policy', {'user_id': 'P-1001'})
    Test 2 (messy spacing): ('check_rules', {...})
    Test 3 (no action): (None, None)

STEP 1.5 — Build agent prompts
FILE: agent/prompts.py
STATUS: [ ] Not done

  What this file does:
  - Defines TOOL_DEFINITIONS list (3 tools with descriptions
    and parameter schemas)
  - Defines SYSTEM_PROMPT string instructing the model to:
    * Reason step by step
    * Use exact format: Thought: ... then Action: tool({...})
    * End with: Verdict: APPROVED/DENIED, Payout: $X, Reasoning: ...
  - Defines TOOL_DEFINITIONS_JSON = json.dumps(TOOL_DEFINITIONS)

STEP 1.6 — Build agent execution loop
FILE: agent/loop.py
STATUS: [ ] Not done

  What this file does:
  - Imports all 3 tools, parser, prompts
  - Defines TOOL_REGISTRY dict mapping names to functions
  - Exposes function: run_agent(claim_text, user_id,
    claimed_amount, client, max_steps=8) -> list of messages
  - Loop: call LLM -> parse action -> execute tool ->
    feed result back -> repeat until Verdict or max_steps
  - Returns full conversation trace as list of
    {"role": ..., "content": ...} dicts
  - CRITICAL: if model outputs "Observation:" itself,
    flag it as hallucination and stop that trace

  Test command after writing:
    python agent/loop.py
  (uses OpenAI API — will cost ~$0.01)
  Expected output:
    Full trace printed to console with 3-4 tool calls
    ending in Verdict: APPROVED or DENIED

STEP 2.1 — Write seed scenarios
FILE: data/seeds.py
STATUS: [ ] Not done

  What this file does:
  - Defines SEED_CLAIMS list of ~15 dicts, each with:
    user_id, claim_text, claimed_amount, expected_verdict
  - Must cover all outcome types:
    * 4x clean approvals (different claim types)
    * 3x denials — claim type not covered
    * 2x denials — policy lapsed/cancelled
    * 2x denials — annual claim limit reached
    * 2x edge cases — amount below deductible (payout=0)
    * 2x edge cases — amount exceeds single claim cap

STEP 2.2 — Write data generation script
FILE: data/generate.py
STATUS: [ ] Not done

  What this file does:
  - For each seed, calls GPT-4o mini to generate
    15 variations (different wording, amounts, circumstances)
  - For each variation, calls run_agent() to get full trace
  - Validates each trace:
    * Must contain >= 3 tool calls
    * Must end with Verdict:
    * Must NOT contain model-generated "Observation:"
    * Verdict must be consistent with tool outputs
  - Saves valid traces to data/train.jsonl
  - Saves rejected traces to data/rejected.jsonl with reason
  - Prints summary: X accepted, Y rejected, reasons breakdown

  Target: 250-300 valid traces in train.jsonl
  Budget: ~$4-6 in OpenAI API calls

  Run command:
    python data/generate.py
  Expected output:
    Generating variations for seed 1/15...
    Running agent on 15 variations...
    Accepted: 12, Rejected: 3 (2 hallucination, 1 short)
    ...
    TOTAL: 267 accepted, 33 rejected
    Saved to data/train.jsonl

STEP 3.2 — Write and run training script
FILE: training/train.py (also paste into Colab)
STATUS: [ ] Not done

  What this script does:
  - Loads Llama-3.2-1B-Instruct from HuggingFace
    (model id: "meta-llama/Llama-3.2-1B-Instruct")
    NOTE: requires HF token and accepting Meta license at
    huggingface.co/meta-llama/Llama-3.2-1B-Instruct
  - Applies LoRA config:
    r=16, lora_alpha=32, lora_dropout=0.05
    target_modules=["q_proj","v_proj","k_proj","o_proj"]
  - Formats each trace using model's chat template
  - Trains for 3 epochs, lr=2e-4, batch_size=4,
    gradient_accumulation=4
  - Saves adapter to: ./insureagent-lora-final/
  - Logs loss curve (should decrease steadily,
    final loss around 0.8-1.2 for this task)

  Watch for during training:
  - Loss should decrease in first 50 steps
  - If loss spikes or plateaus at step 1, stop —
    likely a data formatting issue
  - Training time: ~90 min on T4, ~35 min on A100
